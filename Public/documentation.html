myFlix Documentation

  • This app will allow users to search/access information about movies by name, director, and/ genre. Users will be able to create profile, update profile information, and create a favorites list.

    • Get Movie List

    URL:

  • /movies

    • Method Type:

  • GET

    • Description:

  • Endpoint allows users to get list of all movies and returns an array with list of objects which contains data about movies.

    • Query Parameters:

  • None

    • Expected request data:

  • Required: /movies

    • Example:

  • localhost:8080/movies

    • Expected response data:

  • An array with list of movies containing data about movies and every key is described with value of a string.

    • Example:


    {
    "Title":"Friday",
    "Description":"It's Friday, and Craig(Ice Cube) and Smokey(Chris Tucker) must come up with $200 they owe a local blily or there won't be a Saturday.",
    "Genre": {
    "Name":"Comedy",
    "Description":"A film blends comedy with a life in the hood."
    },
    "Director": {
    "Name":"Felix Gary Gray",
    "Bio":"Felix Gary Gray (born Jliy 17, 1969)[1] is an American film director, film producer, and music video director."
    },
    "ImageURL":"https://en.wikipedia.org/wiki/Friday_(1995_film)#/media/File:Fridayposter1995.jpg",
    "Year":"1995"
    },

    Response status codes:

  • 200: OK
  • 500: Internal server error- Something did not work

    • Get a Genre

    URL

  • /movies/genre/:genreName

    • Method Type:

  • GET

    • Description:

  • This endpoint allows users to search for a genre. Endpoint returns an object with data about specific genre.

    • Query Parameters:

  • :genreName

    • Expected request data:

  • Required: :genreName

    • Example:

  • localhost:8080/movies/genre/Comedy

    • Expected response data:

  • An object containing data about specific genre. Every key is described with value of a string.

    • Example:

    {
    "Name":"Comedy",
    "Description":"Comedies are generally light-hearted dramas and are made to amuse and entertain the audiences."
    },

    Response status codes:

  • 200: OK- JSON object
  • 400: Bad request- Genre not found!
  • 500: Internal server error- Something did not work.

    • Adding a Movie to Favorites

    URL

  • /users/:id/movieTitle

    • Method Type:

  • POST

    • Description:

  • This endpoint allows users to add movie to their favorite list.Endpoint returns an JSON object with users data including updated favorite movies array.

    • Query Parameters:

  • :id, :movieTitle

    • Expected request data:

  • Required: :id, :movieTitle

    • Example:

  • localhost:8080/users/1/movies/New%20Movie

    • Expected response data:

  • A message indicating the movie has been added to the user's array.

    • {
    "Friday has been added to your favorites!"
    }

    Response status codes:

  • 200: OK- Updated users array list
  • 500: Internal server error- Something did not work

    • Removing a Movie from Favorites

    URL

  • /users/:id/:movieTitle

    • Method Type:

  • DELETE

    • Description:

  • This endpoint allows users to remove movie from their list. Endpoint returns an JSON object with users data including updated favorite movies array.

    • Query Parameters:

  • :id, :movieTitle

    • Expected request data:

  • Required: /id, /movieTitle

    • Example:

  • localhost:8080/users/1/movies/Hot%20Rod

    • Expected response data:

  • A message indicating the movie has been removed from the user's array.

    • Example:


    {
    "Friday has been removed from your favorites!"
    }

    Response status codes:

  • 200: OK- Movie has been removed from users list!
  • 500: Internal server error- Something did not work

    • Adding Users

    URL

  • /users

    • Method Type:

  • POST

    • Description:

  • Endpoint allows users to register and returns a JSON object with key values. For key "name" it returns string,for "favorite movies" array, and for an "id" string.

    • Query Parameters:

  • None

    • Expected request data:

  • JSON object with data about user. Required: "name"

    • Example:

    {
    "Name": "Mary",
    }

    Expected response data:

  • JSON object with data about user which includes automatic generated ID

    • Example:

    {
    "Name": "Mary",
    "Email": "mary@email.com,"
    "Password": "password123,"
    "favorite movies": [],
    "id": "8a757bb9-2eec-46bf-86f0-443ff742cf3a"
    }

    Response status codes:

  • 201: Created- New user has been added
  • 400: Bad request- User already exists

    • Updating Usernames

    URL

  • /users/:id

    • Method Type:

  • POST

    • Description:

  • Endpoint allows users to update their username and returns a JSON object with few key values. For key "name" it returns string, for "favorite movies" array, and for an "id" integer.

    • Query Parameters:

  • :id

    • Expected request data:

  • JSON object with data about user.

    • Required:

  • :id

    • Example:

  • localhost:8080/users/1

    • {
    "Username": "newUsername"
    }

    Expected response data:

  • JSON object with data about user which includes,"id" with integer value, "Name", and "favorite movies" with an array.

    • Example:

    {
    "Name": "Slim",
    "Email": "slim@email.com",
    "Password": "password1",
    "favoriteMovies": [],
    "id": 5g890ik9-4vnk-63ed-96f2-678ii889rf4ab,
    }

    Response status codes:

  • 200: OK- New user has been added
  • 500: Internal server error- Something did not work

    • Expected request data:
